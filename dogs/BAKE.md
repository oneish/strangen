# Build System

Strangen uses CMake as its build system.

## Prerequisites

- **CMake** 3.14+
- **C++17 compiler** (tested with g++-12 and clang++-15)
- **External dependencies** cloned as sibling directories (see [Dependencies](#dependencies))

## Quick Start

```bash
# Build and test with GCC
bash gcc.sh

# Build and test with Clang
bash clang.sh

# Build and test with both
bash bake.sh
```

## Dependencies

Clone these into the parent directory of strangen:

```bash
git clone https://github.com/target/libdart.git
git clone https://github.com/microsoft/GSL.git
git clone https://github.com/Tencent/rapidjson.git
git clone https://github.com/chadaustin/sajson.git
git clone https://github.com/doctest/doctest.git
git clone https://github.com/stlab/libraries.git stlab-libraries
```

stlab must be pre-built, see [stlab.sh](../stlab.sh):

```bash
cmake -S ../stlab-libraries -B ../BUILD \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_CXX_COMPILER=/usr/bin/g++-12 \
    -DCMAKE_CXX_STANDARD=17 \
    -DBUILD_TESTING=OFF
cmake --build ../BUILD
```

stlab is not building with clang for some reason. To build without stlab (skipping graph/concurrency targets):

```bash
cmake -B bake -DCMAKE_CXX_COMPILER=clang++-15 -DSTRANGEN_ENABLE_STLAB=OFF
```

## Direct CMake Usage

```bash
# Configure (choose your compiler)
cmake -B bake -DCMAKE_CXX_COMPILER=g++-12

# Build everything
cmake --build bake

# Run tests
cd bake && ctest --output-on-failure

# Build a specific target
cmake --build bake --target test_strange
cmake --build bake --target demo
```

**Note:** The convenience scripts (gcc.sh, clang.sh) use compiler-specific build directories (`bake-gcc/`, `bake-clang/`) for clarity. When using CMake directly, you can use any directory name (e.g., `-B bake`).

## Targets

| Target | Description |
|--------|-------------|
| `strangen` | Code generator tool |
| `demo` | Demo example (with code generation) |
| `example` | Type erasure example (with code generation) |
| `baggage` | Serialization example |
| `currants` | Concurrency example |
| `test_strange` | Core unit tests |
| `test_baggage` | Serialization unit tests |
| `test_graph` | Concurrency unit tests |
| `regenerate_space` | Self-hosting bootstrap (two-pass generation + hash verification) |

## Self-Hosting Bootstrap

The core library header `sauce/strange/space/strange__space.h` is generated by the same framework it defines. The `regenerate_space` target runs a two-pass bootstrap:

1. **Hash**: Record the MD5 hash of the checked-in `strange__space.h`
2. **Pass 1**: Regenerate `strange__space.h` using the current checked-in version, copy to source
3. **Pass 2**: Regenerate again using the pass 1 output, copy to source
4. **Verify**: Compare the hash against the new file — if different, the build fails

Generated intermediates (`.cpp` files) are copied back to source after each pass. This runs automatically as part of every build to surface any changes in the generated output.

## Build Directory Layout

```
bake-gcc/                    # gcc.sh → bake-gcc/, clang.sh → bake-clang/
  sauce/
    generation/
      strangen               # code generator tool
    strange/
      meta/                  # bootstrap intermediates
        pass1_strange__space.h
        pass2_strange__space.h
  snacks/
    demo/                    # demo example + code gen intermediates
    example/                 # type erasure example + code gen intermediates
    baggage                  # serialization example
    currants                 # concurrency example
  taste/
    strange/
      test_strange           # core tests
      implementation/
        baggage/
          test_baggage       # serialization tests
        graph/
          test_graph         # concurrency tests
```
